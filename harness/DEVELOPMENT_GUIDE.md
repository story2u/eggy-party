# Development Guide

## 代码风格

- 使用 TypeScript strict 模式。
- Three.js 对象工厂保持小而清晰，避免单文件堆积过多几何体。
- 重要 Object3D 设置 `name` 和 `userData.kind`，方便测试和后续交互。
- 不添加远程依赖资源来支撑基础场景。
- 不把 3D canvas 放进卡片或营销布局，首屏就是游戏展示。

## TDD 规范

- 新行为先写测试。
- 场景结构、对象类别、动画意图优先写 Vitest。
- 真实浏览器渲染、响应式和 canvas 像素检查写 Playwright。
- 测试名称要描述行为；必要时用注释说明设计意图。
- 改需求时可以改测试，但必须同步 harness 中的功能地图和验证说明。

## Three.js 约定

- renderer 生命周期只在 `EggyIslandApp` 中管理。
- `src/world/` 工厂只负责创建场景对象，不访问 DOM。
- 动画使用 `delta` 和 `elapsed`，不要写死帧率。
- 新对象默认考虑 shadow 设置，但避免过高几何复杂度。
- 后续引入 glTF/纹理时，要提供加载失败 fallback。

## UI 约定

- 不做 landing page。
- 不用可见文字解释快捷键、功能或技术实现。
- UI 如需增加，保持轻量，不遮挡 3D 主体。
- 移动端必须检查尺寸和遮挡。

## 不建议的改动

- 不继续在根目录新增 Python 游戏逻辑。
- 不为了展示阶段引入物理引擎。
- 不引入 React/Vue，除非 UI 状态复杂度明显需要。
- 不把所有场景对象写进 `main.ts`。

## 文档同步规则

代码变化触及以下内容时，必须同步 harness：

- 技术栈或运行命令。
- 场景结构和模块边界。
- 新玩法、新交互、新资源管线。
- 新测试方式或验证命令。
- 长期架构取舍。
